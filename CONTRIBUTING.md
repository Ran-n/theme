## Code of Conduct

We expect all contributors to treat each other with respect. Harassment, discrimination, and bad-faith participation of any kind will not be tolerated.

---

## Reporting Bugs

Open an issue and include:

- A clear, descriptive title
- Step-by-step reproduction instructions
- Expected behaviour vs. actual behaviour
- Your environment (OS, Discord client version, Vencord version)

---

## Suggesting Features

Open an issue with:

- A description of the change and its motivation
- Screenshots or mockups if applicable

---

## Submitting Changes

1. Fork the repository
2. Create a branch off `main`
3. Make your changes
4. Open a pull request against `main` with a clear description

---

## Commit Style

Use Conventional Commits with bracket prefixes:

```
[A] feat: add support for xyz
[F] fix: correct color for abc
[R] refactor: simplify variable definitions
```
